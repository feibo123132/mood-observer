# 认证配置修复计划

## 问题分析
用户遇到的错误提示 **“请联系开发者在身份源列表开启邮箱登录”** 是由腾讯云开发 (CloudBase) 后端返回的明确错误。
这表明虽然前端代码已经实现了“邮箱验证码登录”逻辑，但在 **云开发控制台** 中，该登录方式尚未启用。这是云开发环境的安全默认设置，必须手动开启。

## 解决步骤 (无需修改代码)

### 1. 开启“邮箱登录”权限
请按照以下步骤在腾讯云控制台操作：
1.  登录 [腾讯云开发控制台](https://console.cloud.tencent.com/tcb)。
2.  进入您的环境 **`jieyou-3gr01mvob9ad92de`**。
3.  在左侧菜单栏找到 **“用户管理”** (User Management) 或 **“登录授权”** (Login Authorization)。
4.  在登录方式列表中，找到 **“邮箱登录”** (Email Login)。
5.  点击 **“开启”** (Enable) 或开关按钮。
6.  **重要配置**：在开启配置中，确保勾选或配置 **“验证码登录”** (Verification Code) 相关的选项（如果控制台有区分“链接”和“验证码”，请确保验证码模式可用）。通常开启“邮箱登录”即默认支持验证码。

### 2. 检查“匿名登录” (可选但推荐)
由于 `useAuthStore.ts` 的 `initAuth` 方法中会尝试匿名登录以确保连接：
- 建议同时检查 **“匿名登录”** (Anonymous Login) 是否也已开启。如果未开启，建议开启以提高应用初始化的稳定性。

## 验证
1.  配置完成后，回到应用界面。
2.  刷新页面。
3.  再次输入邮箱并点击 **“发送验证码”**。
4.  此时应该不会再报错，而是进入 60 秒倒计时，并收到验证邮件。

## 代码检查 (已确认无误)
-   前端代码 `auth.getVerification({ email })` 和 `auth.signInWithEmail(...)` 的调用方式完全符合腾讯云 SDK 规范。
-   错误处理机制也能正确捕获并显示来自云端的错误信息（即截图中的红色报错）。
-   因此，**不需要修改任何代码**，只需完成控制台配置即可。
