# 数据迁移计划：旧“随意邮箱” -> 新“真实邮箱” (V2 增强版)

## 问题陈述
用户之前使用“随意填写”的邮箱（如 `1@qq.com`）作为身份标识（UserID）存储了大量数据。
现在系统升级为“真实邮箱验证码登录”，用户将使用真实的邮箱（如 `real@qq.com`）登录。
由于 UserID 变更，用户登录新账号后无法看到旧账号下的数据。

## 解决方案：数据迁移工具 (Data Migration Tool) - 前端增强版

我们需要在应用中提供一个**一次性**的数据迁移功能，允许用户将旧账号（随意邮箱）的数据“复制”到当前登录的新账号（真实邮箱）名下。

## 实施步骤

### 1. 核心迁移逻辑 (`migrateData` 函数)
-   **目标集合**：
    -   `mood_records` (情绪记录)
    -   `mood_reports` (情绪报告)
    -   `socratic_records` (思想手术记录)

-   **执行流程**：
    1.  **查询旧数据**：根据用户输入的 `oldEmail` 查询旧数据。
        -   **CRITICAL**: 必须加上 `.limit(1000)`，否则只会返回 20 条。
        -   `const oldRecords = await db.collection(...).where({ userId: oldEmail }).limit(1000).get()`
    2.  **去重检查 (Critical)**：
        -   对于每一条旧数据，先在新账号下查询是否存在相同的业务 ID (`id`)。
        -   `const exists = await db.collection(...).where({ userId: newEmail, id: oldRecord.id }).count()`
        -   如果 `exists.total > 0`，则跳过该条记录，防止重复迁移。
    3.  **清洗数据**：
        -   从旧数据对象中剔除 `_id` 和 `_openid` 等系统字段。
        -   保留所有业务字段。
        -   将 `userId` 修改为 `newEmail`。
    4.  **写入新数据**：
        -   执行 `add()` 操作。
        -   (可选) 记录迁移成功的条数。
    5.  **后续处理**：
        -   暂时不删除旧数据（为了安全和权限考虑）。
        -   **强制刷新**：迁移完成后，调用 `window.location.reload()` 确保用户立即看到最新数据。

### 2. UI 组件 (`src/components/DataMigrationModal.tsx`)
-   **功能**：一个简单的模态框。
-   **交互**：
    -   输入框：提示“请输入您之前使用的旧邮箱（如 1@qq.com）”。
    -   按钮：“开始迁移”。
    -   进度显示：显示迁移进度（例如 "正在迁移情绪记录... 5/20"）。
    -   结果反馈：迁移成功/失败的提示。
-   **触发入口**：
    -   集成到 `HomePage.tsx` 的侧边栏底部。
    -   按钮文案：“🛠️ 找回旧数据”。

## 验证
1.  登录新账号 `real@qq.com`。
2.  点击“找回旧数据”。
3.  输入旧账号 `1@qq.com`。
4.  观察控制台日志和界面提示，确认迁移过程无报错。
5.  迁移完成后，刷新页面，检查旧数据是否已出现在新账号下。
6.  再次点击迁移，输入相同旧账号，验证去重逻辑是否生效（不应重复添加数据）。
