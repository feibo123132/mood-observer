# 添加音频类型标识与调试日志

根据您的指示，我将对 `src/pages/HomePage.tsx` 进行以下修改，以协助解决 `NotSupportedError` 并验证路径：

1.  **显式指定音频类型**：
    在 `<audio>` 标签中添加 `type="audio/mpeg"` 属性。这有助于浏览器（特别是移动端浏览器）更快地识别音频格式，避免因嗅探失败导致的播放错误。

2.  **增加加载日志**：
    在 `useEffect` 钩子中，当调用 `audioRef.current.load()` 后，立即打印当前加载的音频路径 `activeAudioSrc`。这将帮助我们在控制台中确认实际请求的 URL 是否正确。

### 变更详情

**文件**: `src/pages/HomePage.tsx`

```typescript
// 修改前
<audio 
  ref={audioRef} 
  src={activeAudioSrc} 
  style={{ position: 'absolute', opacity: 0, pointerEvents: 'none', width: 0, height: 0 }} 
  preload="auto"
  playsInline
/>

// 修改后
<audio 
  ref={audioRef} 
  src={activeAudioSrc} 
  type="audio/mpeg" // 新增
  style={{ position: 'absolute', opacity: 0, pointerEvents: 'none', width: 0, height: 0 }} 
  preload="auto"
  playsInline
/>
```

```typescript
// 修改前
useEffect(() => {
  if (activeAudioSrc && audioRef.current) {
    audioRef.current.load();
  }
}, [activeAudioSrc]);

// 修改后
useEffect(() => {
  if (activeAudioSrc && audioRef.current) {
    audioRef.current.load();
    console.log('Loading audio:', activeAudioSrc); // 新增
  }
}, [activeAudioSrc]);
```
